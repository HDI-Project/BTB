bandit(choice_rewards)¶

Return the choice to take next using multi-armed bandit

Multi-armed bandit method. Accepts a mapping of choices to rewards which indicate their historical performance, and returns the choice that we should make next in order to maximize expected reward in the long term.

The default implementation is to return the arm with the highest average score.

Parameters

choice_rewards (Dict[object, List[float]]) – maps choice IDs to lists of rewards.

Returns

the name of the choice to take next.

Return type

str

compute_rewards(scores)[source]¶

Compute the “velocity” of (average distance between) the k+1 best scores. Return a list with those k velocities padded out with zeros so that the count remains the same.

select(choice_scores)[source]¶

Select the choice with the highest best-K velocity. If any choices don’t have MIN_K scores yet, return the one with the fewest.
